# 项目架构与约定

## 解决方案结构

`slnx` 格式，含 3 个项目，目标框架 `net10.0-windows`：

| 项目 | 类型 | 说明 |
|------|------|------|
| `淼喵妙神奇工具库` | Library | 核心逻辑：键鼠库、感知库、输出库、AI 系统 |
| `淼喵妙用户界面` | WinExe | WPF 前端，MVVM 模式（Controls/Dialogs/ViewModels/Data） |
| `淼喵妙测试项目` | Library | xUnit + StaFact 单元测试 |

**引用关系**：用户界面 → 工具库；测试项目 → 工具库 + 用户界面

## Python 环境

项目根目录 `.venv/` 为 Python 虚拟环境，通过 `requirements.txt` 管理依赖。ChromaDB 服务由 C# 应用自动启动（`向量记忆管理器.启动ChromaDB服务()`），用户无需手动操作。

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt  # 安装依赖
.\.venv\Scripts\python.exe -m chromadb run --path chroma_data --port 8000  # 手动启动（通常不需要）
```

## 通用 MCP 工具与 AI 提示词同步

每次修改 [`通用MCP工具.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/通用MCP工具.cs)（增/删/改工具功能），必须同步更新 [`AI配置管理器.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/AI配置管理器.cs) 中 `构建提示词()` 和 `构建消息列表()` 两个方法里的 `【通用MCP工具使用指南】` 隐藏提示词，确保 AI 能正确理解和使用新增/变更的工具。

## 弹窗/通知：强制使用通知工具接口

项目中所有弹窗、输入框、确认框、错误提示、警告提示、吐司通知等用户交互，**必须**通过 [`通知工具.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/输出库/通知工具.cs) 中的静态委托调用，**严禁**绕过接口直接使用 `MessageBox.Show`、`Microsoft.VisualBasic.Interaction.InputBox` 或其他弹窗方式。

### 可用的委托

| 委托 | 签名 | 用途 |
|------|------|------|
| `信息弹窗` | `Action<string>` | 显示信息提示 |
| `输入弹窗` | `Func<string, string, string, string>` | 获取用户文本输入（prompt, title, defaultResponse → result） |
| `吐司通知` | `Action<string>` | 右下角浮动通知，3秒自动消失 |
| `确认弹窗` | `Func<string, bool>` | 是/否确认对话框 |
| `错误弹窗` | `Action<string>` | 显示错误信息 |
| `警告弹窗` | `Action<string>` | 显示警告信息 |
| `选项弹窗` | `Func<string, List<string>, string>` | 单选选项对话框 |
| `图片查看器` | `Action<string, List<Bitmap>>` | 多图浏览弹窗 |

### 调用规范

```csharp
// 工具库中（所有委托可能为 null，必须用 ?.Invoke + ?? 兜底）
通知工具.信息弹窗("操作完成");
通知工具.输入弹窗?.Invoke("请输入名称:", "输入", "") ?? "";
通知工具.错误弹窗?.Invoke($"失败: {ex.Message}");
通知工具.确认弹窗?.Invoke("确认删除？") ?? false;

// 用户界面中（委托在 App.OnStartup 中已初始化，可为 null 的场景也建议用 ?.Invoke）
通知工具.警告弹窗("请选择脚本文件");
```

### 初始化位置

所有委托的实际实现由 [`App.xaml.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/App.xaml.cs) 在 `OnStartup` 中统一注入，底层使用 WPF 标准 `MessageBox.Show` 和 `Microsoft.VisualBasic.Interaction.InputBox`。

## 全局变量数据类型

脚本运行时可在全局字典 `Dictionary<string, object>` 中存取变量，**仅限以下类型**：

- 整数：`int`
- 浮点数：`float`、`double`
- 字符串：`string`
- 布尔值：`bool`
- 坐标：`System.Drawing.Point` 或 `System.Windows.Point`
- 尺寸：`System.Drawing.Size` 或 `System.Windows.Size`
- 矩形：`System.Drawing.Rectangle` 或 `OpenCvSharp.Rect`
- 图片：`OpenCvSharp.Mat` 或 `System.Drawing.Bitmap`
- 列表：`List<T>`（`T` 仅可为以上类型之一）

> 注意：`Point`、`Size` 在 `System.Drawing` 和 `System.Windows` 两个命名空间中各有一个版本，`Rect` 在 `System.Drawing` 和 `OpenCvSharp` 中各有定义。编写节点代码时应注意区分，或使用 `using` 别名。
