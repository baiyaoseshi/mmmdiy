# 节点系统概述

## 节点基类

所有功能节点继承 [`控制节点`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/控制节点.cs)，它实现了 `INotifyPropertyChanged`。

### 子类必须实现的方法

| 方法 | 类型 | 说明 |
|------|------|------|
| `节点动作(IntPtr hWnd, Dictionary<string,object> 全局)` | 实例方法 | 核心执行逻辑，返回 `true`=成功跳转，`false`=失败跳转 |
| `保存为字符串(自动任务脚本)` | 实例方法 | 序列化为 `键[值],` 格式字符串 |
| `解析(string, IntPtr)` | 静态方法 | 从字符串反序列化，返回节点实例 |
| `创建节点(IntPtr)` | 静态方法 | 工厂方法，创建带默认值的节点实例 |

### 可选覆写的成员

| 成员 | 类型 | 说明 |
|------|------|------|
| `需要窗口句柄` | 属性 | 默认 `true`。设为 `false` 的节点可在无绑定窗口时执行 |
| `初始化()` | 实例方法 | 有时需要覆写，例如节点需要从列表中按序读取数据 |

### 基类提供的通用字段

`节点名字`、`节点备注`、`成功后跳转`/`失败后跳转`（链表指针，指向节点在列表中的索引）、`条件列表`（前置**联想器**列表，执行前逐一检查，任一不满足则跳过此节点）。这些由 `控制节点.保存为字符串` 和 `解析基类字段` 统一处理，子类无需关心。

## 序列化格式

脚本头部：`进程名[xxx]窗口标题[xxx]脚本备注`
每个节点：`@节点标签名:字段1[值1],字段2[值2],...`

跳转关系用节点在列表中的索引表示，`-1` 表示无跳转（即停止）。

## 执行模型

**链表式遍历**：从起始索引开始，每个节点执行 `节点动作()` → 根据返回值选择 `成功后跳转` 或 `失败后跳转` 的下一个节点。若跳转为 `null`，则顺序执行下一个。遇到 `停止任务` 类型节点则终止。

## 添加新功能节点的步骤

### 1. 创建节点类文件

在 `键鼠库\动作\流程节点\` 下对应子目录中创建 `.cs` 文件，遵循以下约定：

- 命名空间：`淼喵妙神奇工具库.键鼠库.动作.流程节点`
- 继承 `控制节点`
- 必须覆写的实例方法：`节点动作`、`保存为字符串`
- 必须实现的静态方法：`解析`、`创建节点`
- 可根据需要覆写：`需要窗口句柄`（属性，默认 `true`）、`初始化()`（例如节点从列表中按序读取时需要）

- 参考同目录下其他节点（如 `空节点.cs`、`显示变量.cs`）的文件结构和格式

### 2. 注册：反序列化工厂

在 [`自动任务脚本.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/自动任务脚本.cs) 的 `解析()` 方法中的 `switch` 语句里，添加新节点的 case，将标签映射到 `解析()` 静态方法：

```csharp
case "新节点标签名":
    return 新节点类名.解析(字符串, hWnd);
```

### 3. 注册：UI 分类

在 [`NodeSelectionDialog.xaml.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/Dialogs/NodeSelectionDialog.xaml.cs) 的 `GetCategory()` 方法中，将新节点类型名添加到对应分类的 if 条件中，例如：

```csharp
|| name == "新节点类名"
```

### 4. 编译验证

分别编译两个项目确认无错误：

```powershell
dotnet build "淼喵妙DIY应用程序\淼喵妙神奇工具库\淼喵妙神奇工具库.csproj"
dotnet build "淼喵妙DIY应用程序\淼喵妙用户界面\淼喵妙用户界面.csproj"
```

## 6 个节点分类

| 分类目录 | 说明 |
|----------|------|
| `指令与流程控制/` | 流程跳转、变量显示、PowerShell、计算等 |
| `按键输入/` | 键盘按键相关操作 |
| `鼠标输入/` | 鼠标操作（含点击图片/文字等复合操作） |
| `判断与读取/` | 屏幕识别、截图、Excel读取等 |
| `应用与窗口/` | 打开/绑定应用、窗口调整 |
| `其他/` | 兜底分类 |
