# 脚本自动管理系统

## 架构概览

脚本自动管理系统分为三个层面：

| 层面 | 位置 | 职责 |
|------|------|------|
| **执行引擎** | 工具库 `键鼠库/动作/` | 节点执行、脚本序列化/反序列化、任务控制 |
| **调度层** | 用户界面 `ViewModels/MainWindowViewModel.cs` | 定时器驱动、队列管理、任务协调 |
| **持久化** | 用户界面 `Data/` + `Services/` | 用户数据自动保存、脚本文件读写 |

## 一、执行引擎

### 1.1 脚本链表遍历执行

[`自动任务脚本.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/自动任务脚本.cs) 包含脚本的容器和执行逻辑。

**`执行(int 起始节点索引)` 方法流程**：

1. 所有节点调用 `初始化()`
2. 从 `起始节点索引` 开始进入 `while(true)` 循环
3. 每个节点执行 `节点.执行(hWnd, 全局)` ——内部先检查 `条件列表`（前置联想器），再执行 `节点动作()`
4. 根据返回值选择跳转：
   - `true` → `成功后跳转`（链表索引）
   - `false` → `失败后跳转`（链表索引）
5. 若跳转为 `null`（索引 `-1`），则顺序执行下一个节点
6. 遇到 `停止任务` 类型节点则 `return false` 终止
7. 执行到最后一个节点（无后续）则 `return true` 完成

### 1.2 取消与暂停机制

[`任务控制管理器.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/任务控制管理器.cs) 是单例控制器：

| 方法/属性 | 说明 |
|-----------|------|
| `取消()` | 设置 `_isCanceled = true`，脚本执行循环会抛出 `OperationCanceledException` |
| `暂停()` / `继续()` | 控制 `_isPaused` 标志 |
| `重置取消()` | 每次执行前调用，将 `_isCanceled` 重置为 `false` |
| `触发插队请求(string)` | 引发 `插队请求` 事件，由 MainWindowViewModel 处理 |

在 [`控制节点.等待(int 毫秒)`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/控制节点.cs) 方法中：
- 循环检测 `任务控制管理器.实例.IsCanceled` → 抛出 `OperationCanceledException`
- 循环检测 `任务控制管理器.实例.IsPaused` → 暂停等待（`Thread.Sleep(5)` 自旋）

### 1.3 序列化格式

脚本头部：`进程名[xxx]窗口标题[xxx]脚本备注`

每个节点：`@节点标签名:字段1[值1],字段2[值2],...`

跳转关系用节点在列表中的索引表示，`-1` 表示无跳转（即停止）。

反序列化在 [`解析()`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/自动任务脚本.cs) 方法中，使用 `AsParallel()` 并行解析所有节点，提升性能。

## 二、任务调度体系

调度中心是 [`MainWindowViewModel.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/ViewModels/MainWindowViewModel.cs)（约 1200+ 行），维护三类定时器和两种任务列表。

### 2.1 三种系统定时器

| 定时器 | 间隔 | 回调 | 职责 |
|--------|------|------|------|
| `_schedulerTimer` | 1秒 | `CheckScheduledTasks` | 检查定时任务是否到期 |
| `_countdownTimer` | 0.1秒 | `UpdateCountdowns` | 更新 UI 倒计时和节点运行状态 |
| `_autoSaveTimer` | 5秒 | `AutoSaveUserData` | 自动保存用户数据到磁盘 |

### 2.2 定时任务（ScheduledTask）

[`ScheduledTaskViewModel.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/ViewModels/ScheduledTaskViewModel.cs) 支持四种调度模式：

| 模式 | 常量 | 行为 |
|------|------|------|
| **绝对时间** | `调度模式.绝对时间` | 在指定时间点执行一次，完成后自动禁用 |
| **倒计时** | `调度模式.倒计时` | 从现在起 N 秒后执行一次，完成后自动禁用 |
| **重复** | `调度模式.重复` | 每天/每周/每月 的指定时间执行，执行后自动计算下次触发时间 |
| **循环** | `调度模式.循环` | 每隔 N 秒循环执行，永不停止（除非手动禁用） |

**到期检查**（`CheckScheduledTasks`，每秒执行）：
1. 遍历 `ScheduledTasks` 列表
2. 当 `ScheduledDateTime <= DateTime.Now` 时触发
3. 调用 `AddTaskToQueue()` 加入任务队列
4. 根据调度模式决定：禁用任务 or 重新计算下次时间

### 2.3 触发任务（TriggerTask）

[`TriggerTaskViewModel.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/ViewModels/TriggerTaskViewModel.cs) 支持两种触发类型：

| 类型 | 触发条件 |
|------|----------|
| **窗口出现** | 当指定标题的窗口出现时触发 |
| **指令** | 通过远程 HTTP 指令 `/task 任务名` 触发 |

## 三、任务队列管理

### 3.1 两层队列架构

**第一层：`MainWindowViewModel.TaskQueue`**（`ObservableCollection<TaskItemViewModel>`）
- UI 线程管理，用于显示任务列表
- 来源：手动运行、定时任务到期、触发任务激活、脚本运行、手动添加文件

**第二层：`任务控制管理器.运行队列`**（`ConcurrentQueue<自动任务脚本>`）
- 单例线程安全 FIFO 队列
- 当前未被 `ProcessQueue` 实际使用（该方法直接调用 `firstTask.Script.执行()`），但保留作为独立调用通道

### 3.2 FIFO 队列处理

`ProcessQueue()` 方法：
1. 检查 `_isProcessingQueue`（防重入）
2. 检查 `_queuePaused`（插队期间暂停）
3. 取出第一个状态为"队列中"或"等待中"的任务
4. `Task.Run` 中执行脚本
5. 执行完成后同步窗口句柄等信息回页面
6. `finally` 中递归调用 `ProcessQueue()` 继续处理下一个

### 3.3 插队机制

由 [`插队.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/键鼠库/动作/流程节点/指令与流程控制/插队.cs) 节点触发，流程：

```
插队节点 → 任务控制管理器.触发插队请求("脚本文件路径")
  → MainWindowViewModel.处理插队请求:
      1. 设置 _queuePaused = true  (冻结普通队列)
      2. 取消当前正在执行的任务
      3. 将插队脚本 Insert(0, ...) 到 TaskQueue 最前端
      4. 恢复 _queuePaused = false
      5. 触发 ProcessQueue()
```

### 3.4 任务状态流转

```
队列中 → 运行中 → 完成 / 失败（取消/异常）
等待中 → 运行中 → 完成 / 失败（取消/异常）
```

[`TaskConstants.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/TaskConstants.cs) 定义了五个状态：`队列中`、`等待中`、`运行中`、`完成`、`失败`。定时任务初始为"等待中"，手动任务和脚本运行为"队列中"。

## 四、脚本执行入口汇总（11 种）

| 入口 | 说明 | 是否入队列 |
|------|------|------------|
| **手动运行** | 用户点击"运行"按钮 | 是 |
| **从此运行** | 从指定节点索引开始执行 | 是 |
| **单节点运行** | 仅执行一个节点 | 是 |
| **支线运行** | 后台 `Task.Run` 独立执行 | **否** |
| **定时任务到期** | `CheckScheduledTasks` 检测到时间到 | 是 |
| **触发任务激活** | 远程指令 `/task` 或窗口出现 | 是 |
| **手动添加任务** | 文件对话框选择 `.script` 文件 | 是 |
| **嵌套执行** | `执行任务` 节点调用另一个脚本（同步） | **否** |
| **智能执行** | `智能规划任务` 通过 AI 匹配脚本 | **否**（在节点内同步执行） |
| **插队执行** | `插队` 节点插入队列最前端 | 是（插到队首） |
| **MCP 工具调用** | AI 通过 `MCP工具管理器.执行工具()` 调用 | **否** |

> 关键区分：**入队列**的任务串行执行，**不入队列**的支线/嵌套执行是独立线程，可能与队列中的任务并发运行。

### 5.1 执行任务节点

[`远程指令服务器.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙神奇工具库/远程指令服务器.cs)（约 270 行）基于 `HttpListener`。

|----------|------|------|
| `/2AI <对话标签> <消息>` | `AI指令` | 触发 AI 对话 |

## 七、持久化

### 7.1 用户数据

[`UserData.cs`](file:///d:/D/淼喵妙脚本DIY应用程序/淼喵妙DIY应用程序/淼喵妙用户界面/Data/UserData.cs) 是用户数据的根模型，JSON 序列化后存储于 `%AppData%/淼喵妙脚本DIY/userdata.json`。

主要包含：
- `脚本页面列表`：所有打开的脚本页签
- `分类列表`：脚本分类及其包含的脚本文件路径
- `定时任务列表`：所有定时任务配置
- `触发任务列表`：所有触发任务配置
- `远程指令配置`：端口、令牌
- `窗口透明度` 等 UI 设置

### 7.2 自动保存

`_autoSaveTimer` 每 5 秒触发 `AutoSaveUserData()`，将 `UserData` 序列化为 JSON 写入磁盘。数据变更时通过 `IsUserDataDirty` 标记，避免无变化时写入。
