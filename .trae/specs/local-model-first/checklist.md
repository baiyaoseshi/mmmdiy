## 移除旧代码 — 配置管理器 + UI

## MCPToolDefinition

## 脚本 GUID
- [ ] 脚本头部含 `脚本Id[GUID]`，旧格式兼容
- [ ] 工具ID = `tool_` + GUID 前12位，改名不变

## 共享印象 — 数据模型
- [ ] `工具印象`/`印象修正记录`/`印象存储根` 含全部字段
- [ ] `ai_tool_impressions.json` 线程安全读写

## 共享印象 — 生命周期
- [ ] 新工具 → 初始印象（名称+备注，置信度 0.3）
- [ ] 首次成功 → 第一印象（置信度 0.7），覆盖初始
- [ ] 一致 → 置信度 +0.02（上限 0.95）
- [ ] 偏差 → 追加修正，第一印象不变
- [ ] 矛盾 → 覆盖第一印象，置信度重置 0.5，旧印象留历史
- [ ] 所有变更立即对全部 AI 可见

## 共享印象 — 融入工具描述
- [ ] 置信度 ≥ 0.5 → 描述附加 `【印象】...（置信度: X%）`
- [ ] 置信度 < 0.5 → 不增强
- [ ] `构建工具定义列表()` 统一查询印象，所有 AI 路径一致生效

## 经验模型评估
- [ ] Prompt 含当前印象，输出 `quality` + `impression_match`
- [ ] 30 秒超时，异常不阻塞

## 训练数据 + 训练管线
- [ ] 训练样本基于 `工具调用记录` 扩展，保留 `调用时计划树`、`调用后树修改`、`是否成功`、`耗时ms`
- [ ] 新增字段：`quality`、`impression_match`、`工具GUID`、`印象版本`
- [ ] JSONL `input` 以印象为主体，**不含**用户名称/备注/分类路径
- [ ] 初始印象（置信度 0.3）的调用不入训练数据
- [ ] 数据清洗、评测集、SFT/DPO/RL TODO 占位

## 训练触发与 AI 锁定

- [ ] 训练完成/失败 → 自动解除锁定
